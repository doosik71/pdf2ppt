# 프롬프트

프로젝트 이름은 PDF2PPT이다.
AI가 과제 제안서, 보고서, 논문 등의 PDF 문서를 분석 및 이해하고 PPT 발표자료를 작성하는 프로그램이다.
챗봇 모델로 Gemini 또는 Ollama 등을 사용할 수 있다.
Gemini 모델을 사용하면 환경변수로부터 API key를 받는다.
Ollama 모델을 사용하면 환경변수로부터 URL 주소와 모델 id를 받는다.
Stateless 정적 웹 서버로 배포할 예정이다.
API key, Ollama 서버 URL, 모델 id 등의 환경변수는 서버에서만 접근이 가능하며 클라이언트에서는 접근이 금지되어야 한다.
SvelteKit과 tailwind sss를 사용한다.
npmp로 패키지를 관리한다.
GUI는 웹 페이지 화면이며 SPA(단일 페이지 웹 어플리케이션)이다.
단일 화면의 주요 구성은 다음과 같다.

1. PDF 문서를 업로드하는 부분
2. 요약 버튼을 누르면 문서의 요약 내용을 추출하고 보여주는 부분
3. 목차 생성 버튼을 누르면 문서의 내용을 분석하고 목차를 생성한 후 목차를 보여주는 부분
   - 각각의 목차는 슬라이드 한 페이지에 대응한다.
   - 동일 제목의 슬라이드를 여러 장 구성할 경우에는 목차 항목의 끝에 일련 번호를 부여한다.
4. 생성할 슬라이드의 사전정의된 색상 테마를 선택하는 부분
5. 생성할 슬라이드의 사전정의된 스타일(화자의 성향, 청자의 대상, 발표 목적에 따라 다름)을 선택하는 부분
6. 목차에서 생성할 페이지를 선택하는 부분
   - 슬라이드 전체를 한꺼번에 생성하는 것이 아니라 목차 항목에 따라 한 페이지씩 생성한다.
7. 생성된 슬라이드 화면을 보여주는 부분
8. 슬라이드 결과물을 사용자가 평가하는 부분
   - 슬라이드가 마음에 든다
   - 슬라이드가 마음에 들지 않는다.
     1. 스타일 부적합
     2. 내용 부적합
     3. HTML 형식 깨짐
9. 생성된 슬라이드를 PNG 또는 HTML(SVG 포함)로 export하는 버튼

이 프로그램이 PPT 슬라이드를 생성하는 방법은 다음과 같다.

1. 앱은 챗봇에게 다음과 같은 프롬프트 및 컨텍스트를 제공한다.
   - 과제의 목적을 정의한 프롬프트
   - 작성해야 할 슬라이드의 색상 테마와 스타일을 기술한 프롬프트
   - 문서의 전체 내용
   - 문서의 주요 요약 내용
   - 문서의 전체 목차
   - HTML 템플릿 (이 템플릿은 사용자가 선택한 색상 테마에 따라 사전 정의된다)
   - HTML 템플릿 중 특정 토큰(예를 들어 "[SLIDE_CONTENTS_HERE]"와 "[SLIDE_CSS_HERE]")의 위치에 생성을 해야 한다고 지시하는 프롬프트
   - 문서의 전체 목차 중 슬라이드 페이지를 생성해야 할 선택된 목차 항목
2. 챗봇 엔진이 대화 이력을 관리하는 경우에는 토큰의 절약을 위해 다음의 내용 전달을 매번 보내지 않고 생략할 수 있다.
   - 과제의 목적을 정의한 프롬프트
   - 작성해야 할 슬라이드의 색상 테마와 스타일을 기술한 프롬프트
   - 문서의 전체 내용
   - 문서의 주요 요약 내용
   - 문서의 전체 목차
   - HTML 템플릿 (이 템플릿은 사용자가 선택한 색상 테마에 따라 사전 정의된다)
   - HTML 템플릿 중 특정 토큰(예를 들어 "[SLIDE_CONTENTS_HERE]"와 "[SLIDE_CSS_HERE]")의 위치에 생성을 해야 한다고 지시하는 프롬프트
3. 그러나 챗봇이 대화 컨텍스트를 잊고 있다고 판단되면(예: 스타일 부적합, 내용 부적합, HTML 형식 깨짐) 필요한 프롬프트와 컨텍스트를 다시 전송해야 한다.

지금까지의 내용을 숙지 및 검토하고 상세한 개발 계획을 수립하라.
수립된 개발 계획을 DEVELOPMENT_PLAN.md 파일에 작성하라.

DEVELOPMENT_PLAN.md 문서를 검토하고 README.md 파일을 만들어라.
README.md 파일에는 개발환경 구축 안내도 포함해야 한다.

프로젝트 생성부터 시작해서 단계별 개발 진행 과정을 TODO.md 파일에 작성하라.

TODO.md 파일의 해야할 일 항목을 코딩 에이전트에 전달할 명령문으로 변환하라.
TODO 항목 한줄이 명령문 한줄로 변환되어야 한다.
변환된 명령문들을 COMMANDS.md 파일에 저장하라.

SvelteKit 프로젝트를 생성하라.

Tailwind CSS 설정 및 전역 스타일 초기화를 진행하라.

필수 패키지 설치(`zod`, PDF 파서, LLM SDK/클라이언트, 테스트 도구)를 진행하라.

이제 기본 아키텍처 골격 구현 단계이다. 디렉터리 구조 생성을 진행하라.

서버 전용 환경변수 로더를 구현하라.
